# 第一章 预备知识

## `C`语言编程原理

面向过程（Procedural Programming）是一种计算机编程范式，它将计算机程序视为一系列可重用的程序单元或子过程的集合，每个子过程都是执行一个特定任务的连续的一系列指令。面向过程编程主要关注如何通过分解问题为更小的子问题来解决问题，强调过程和数据的耦合度。面向过程编程语言如 C，Fortran 等，也可以通过很多编程语言的过程语言编写。

在面向过程编程范式中，程序的控制流程通常是线性的，即依次执行各个子过程，逐步完成整个程序的执行。由于每个子过程都是独立和可重用的，因此可以在不同的程序中使用相同的子过程，从而提高程序的可维护性和可重用性。同时，由于面向过程编程更接近于硬件执行的方式，因此在对计算机资源的使用上更加高效，执行速度也更快。

## 面向对象编程

面向对象（*Object-Oriented Programming，OOP*）是当前主流的编程范式之一，它将数据和操作数据的方法封装在一起，形成对象，对象之间通过消息传递来实现协作和通信，从而组成程序的整体结构。此外，面向对象编程还提供了封装、继承、多态等重要特性，可以更加有效地组织和复用代码，并且对于大型、复杂、多人合作的项目，也提供了更加灵活和可靠的开发方式。

面向对象编程尽可能地将系统中的事物抽象成对象，并将对象之间的关系通过继承、组合等方式进行组合和多态实现，从而形成一个灵活、容易扩展和维护的系统。它强调数据和操作数据的行为，推崇数据和方法的封装，从而保证系统安全性和可维护性。

面向对象编程的优点包括：代码可重用性高、代码维护性高、易扩展、易维护、易理解、可维护性好等。常见的面向对象编程语言有Java、C++、Python、C#等。虽然面向对象编程对初学者来说可能难度较高，但是掌握面向对象编程的知识和技能可以帮助程序员更好地理解和构建大型复杂程序和系统。

## `C++`和泛型编程

泛型编程（*generic programming*）是C++支持的另一种编程模式。
它与`OOP`的目标相同，即使重用代码和抽象通用概念的技术更简单。不
过`OOP`强调的是编程的数据方面，而泛型编程强调的是类型。它们的侧重点不同。
OOP是一个管理大型项目的工具，而泛型编
程提供了执行常见任务（如对数据排序或合并链表）的工具。术语泛型
（generic）指的是创建独立于类型的代码。C++的数据表示有多种类型
——整数、小数、字符、字符串、用户定义的、由多种类型组成的复合
结构。泛型编程需要对语言进行扩展，以便**可以只编写一个泛
型（即不是特定类型的）函数，并将其用于各种实际类型**。

## 程序编译过程

编写了`C++`程序之后，它是如何执行的呢？具体的步骤如下：

1. 在某个文件中编写源代码。
1. 编译源代码。将源代码翻译为机器语言，包含了翻译之后的程序的文件就是程序的目标代码。
1. 将目标代码与其他代码链接起来。

## 编译和链接

`Linux`系统中最常用的编译器是`g++`，`Linux`的多数版本都包括该编译器，但并
不一定总会安装它。`g++`编译器的工作方式很像标准UNIX编译器。

例如，下面的命令将生成可执行文件a.out：

```bash
g++ test.cpp
```

编译多个源文件：

```bash
g++ test.cpp test1.cpp test2.cpp
```
