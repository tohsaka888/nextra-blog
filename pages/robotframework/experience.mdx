# 使用体验

优点：

1. 优秀的 IDE 集成，能够非常方便的与常用 IDE 进行集成，例如：VSCode、PyCharm（官方油管示例使用）
1. 方便模块化，能够通过`Resource`文件方便的将测试用例中公用的部分，比如关键词、变量，抽离并进行模块化，供多个
   测试用例之间使用
1. 可以生成详细的测试报告，其中包含测试用例的执行结果、错误信息和日志等。这些报告易于阅读和理解，可以帮助测试团队更好地了解测试结果，并快速识别和解决问题。

缺点：

1. 创建一个机器人的时间较长（3-5min）
1. 及其不好用的测试用例语法（类似 yml，但是比 yml 还要没条理，只要有两个空格以上的空间都承认为合格的语法），
   这个缺点可能会影响到后续测试用例编写的维护，需要一个团队认可的格式化方案。
1. 暂且没有在官方文档中找到有关嵌入式集成测试的方案，但是似乎可以继承 python 脚本进行测试，如果可以通过 python 脚本
   来进行测试，我认为使用这个框架是可行的
